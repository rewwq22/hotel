# CLAUDE.md — Автоматическая инструкция для каждой сессии
# Этот файл читается Claude Code АВТОМАТИЧЕСКИ при каждом открытии проекта.

## ПЕРВОЕ ДЕЙСТВИЕ ПРИ СТАРТЕ СЕССИИ

**ОБЯЗАТЕЛЬНО** выполни эти шаги прежде чем делать что-либо ещё:

1. Прочитай файл `.claude/state/wip.md` — там текущее состояние незавершённых задач
2. Прочитай файл `AGENT.md` — там все правила проекта, шаблоны и ограничения
3. Если в `wip.md` есть незавершённые задачи со статусом `IN_PROGRESS` или `BLOCKED`:
   - Сообщи пользователю: "Обнаружены незавершённые задачи от предыдущей сессии: [список]"
   - Спроси: продолжить их или начать новое
4. Если `wip.md` пуст или все задачи `DONE` — работай как обычно

## ПРОТОКОЛ РАБОТЫ С ЗАДАЧАМИ (WIP-ТРЕКЕР)

### Перед началом любой работы:
```
Обнови .claude/state/wip.md — добавь задачу со статусом IN_PROGRESS
```

### Каждые 3-5 шагов (или перед тяжёлой операцией):
```
Обнови .claude/state/wip.md — запиши промежуточный прогресс
```

### После завершения задачи:
```
Обнови .claude/state/wip.md — переведи задачу в DONE, добавь результат
```

### При ошибке / блокировке:
```
Обнови .claude/state/wip.md — переведи в BLOCKED, запиши причину
```

## ФОРМАТ ФАЙЛА wip.md

```markdown
# WIP — Work In Progress (обновлено: YYYY-MM-DD HH:MM)

## Текущие задачи

### [СТАТУС] Название задачи
- **Начата:** дата-время
- **Что сделано:** список выполненных шагов
- **Что осталось:** список оставшихся шагов
- **Файлы затронуты:** список путей
- **Контекст:** любая важная информация для продолжения
- **Блокер:** (если BLOCKED) описание проблемы

## Завершённые задачи (последние 5)
### [DONE] Название — дата
- Результат: краткое описание
```

Статусы: `IN_PROGRESS`, `BLOCKED`, `DONE`

## КРИТИЧЕСКИЕ ПРАВИЛА ПРОЕКТА

1. **НЕ ТРОГАТЬ** Яндекс.Метрику (ID 106791450) в `<head>` всех HTML-страниц
2. **НЕ УДАЛЯТЬ** файл подтверждения Яндекс.Вебмастера в корне сайта
3. **НЕ УДАЛЯТЬ** `mail.php` на сервере
4. Все пути в HTML — **относительные** (не от `/`)
5. Подробные правила, шаблоны и структура → в `AGENT.md`
6. Семантическое ядро → в `semantic-core.md`
7. Карта сайта → в `STRUCTURE.md`
8. Деплой → в `DEPLOYMENT.md`
